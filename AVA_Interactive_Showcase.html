1. vector_fifo.sv

Command Elasticity FIFO decouples the core CV32E40X instruction fetch from vector execution, absorbing structural pipeline stalls.

2. vector_lsu.sv (DROM)

Weights are tightly coupled directly near the ALUs via localized DROM, eliminating massive OBI bus contention during matrix loads.

3. pe_32b.sv (vpdot.vv)

Processing Elements execute the custom `vpdot.vv` inline assembly, firing 4 concurrent INT8 Multiplies-Accumulates inside a single 32-bit datapath.

4. 2-Stage Vector Pipeline

Decoupled decode (`vector_decoder.sv`) and arith (`arith_stage.sv`) stages ensure high maximum clock frequency scaling.

Cycle-Accurate Timing Simulation

A WaveDrom-rendered equivalent of a GTKWave simulation trace. This visualizes the cycle-by-cycle decoupling achieved by the FIFO Buffer, the DROM bypassing the OBI bus, and the vpdot pipeline execution.

APU Request & Grant

The CPU fires an instruction to the APU. The FIFO immediately asserts apu_gnt_o, allowing the CPU to unblock instantly.

FIFO Absorption

Because the queue takes the instruction, the CPU continues scalar execution while the AVA pipeline pulls the instruction via fifo_pop later.

DROM Hit (Zero Contention)

The Vector Load Store Unit accesses the DROM. drom_hit goes high while obi_bus_req stays low, saving main memory bandwidth.

Parallel Execution

The 4-Lane PEs do the math (pe_active) and subsequently write the accumulated 8-bit output back to the Vector Register File (vreg_we_o).
